PageTurner Data Project

Analytical architecture on Microsoft Fabric with Lakehouse, Bronze â†’ Silver â†’ Gold layers, Dataflow Gen2 for data processing and Spark notebooks for dimensional modeling.

Bronze

Raw

We created the Lakehouse and loaded system extractions (CSV/Parquet) into the Bronze layer, preserving the fidelity of source data for auditing and reprocessing.

  • Ingestion via upload/OneLake
  • Basic partitioning by period
  • Schema on read

Silver

Curated

We used Dataflow Gen2 to clean, standardize, and enrich data, producing trusted tables in the Silver layer.

  • Normalization of types and dates
  • Handling nulls and duplicates
  • Business rules applied

Gold

Semantic

With Spark notebooks, we created the dimensional model (e.g., dim_books) and fact tables for analytics and BI.

  • Dimensions: dim_books, dim_authors, dim_clients
  • Facts: fact_sales
  • Surrogate keys and SCD (when applicable)

Stack and Integrations

Fabric Lakehouse, Dataflow Gen2, Apache Spark (notebooks), and Django + Ninja to expose the catalog API. The goal is to enable analytical exploration, pipeline testing, and integration with modern applications.
